User Flow — Problemas y Fixes

3 problemas concretos encontrados analizando el código

Flujo actual — confuso
📋
Job Board → "ATS Evaluator"
User subió CV, tiene match scores
🔴 Problema 1 — Al llegar al CV Evaluator, tiene que subir el CV otra vez. El backend puede reutilizar la sesión, pero el frontend no lo aprovecha.
🔗
Job Detail → "See how to improve your match"
Llega a index.html?job_id=123
🔴 Problema 2 — El job context badge es texto tiny de 12px. El user no sabe para qué puesto está evaluando.
📄
Sube CV + Analyze
Dos clics separados: select file → botón "Analyze CV"
📊
Ve resultados
Score, keywords, recomendaciones
🔴 Problema 3 — Dead end. El único CTA es "← Analyze another CV". No hay forma de volver al job board o aplicar a un puesto.
❓
¿Y ahora qué?
El user no sabe qué hacer después.
Flujo propuesto — claro
📋
Job Board → "ATS Evaluator"
User ya tiene CV session
✅ Fix 1 — Detectar session token en localStorage. Si existe: mostrar chip "Using CV: [filename]" con opción "Change". El user no tiene que re-subir.
🔗
Job Detail → "See how to improve"
Llega con ?job_id=123
✅ Fix 2 — Reemplazar badge tiny por un panel de contexto visible: fondo con color, título del job y empresa destacados.
📄
Sube CV (o usa el existente)
Si tiene sesión activa, el botón "Analyze" aparece directo.
📊
Ve resultados
Score, keywords, recomendaciones
✅ Fix 3 — Agregar CTA al final de los resultados: botón "Browse matching jobs →" que lleva al job board.
✅
El user sabe qué hacer
Ve el score, entiende qué mejorar, puede ir al job board a buscar roles que matcheen.
Los 3 fixes — resumen
1
Subir CV dos veces si ya lo hiciste en el job board
Detectar session token → mostrar chip con el CV activo. Solo pedí re-upload si quiere otro.
2
El badge de contexto de job es invisible (texto tiny)
Panel de contexto prominente arriba del dropzone con título del puesto destacado.
3
Dead end después de ver resultados — no hay próximo paso
CTA "Browse matching jobs →" al pie de los resultados. Cierra el loop job board ↔ evaluator.
