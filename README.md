## The Problem

75% of resumes are rejected by Applicant Tracking Systems before a human ever reads them. Companies use ATS to filter candidates by keyword matching, structured parsing, and formatting rules ‚Äî most job seekers don't know why their applications go unanswered.

**CV Evaluator** automates the diagnosis: upload your resume and receive a scored report with found/missing keywords, formatting issues, and actionable recommendations ‚Äî all generated by Claude via structured outputs.

### Design decisions

- **`src/` vs `backend/` separation** ‚Äî the HTTP layer has no knowledge of LLM logic; the business logic has no knowledge of HTTP. This lets you swap FastAPI for another framework without touching the AI pipeline.
- **Prompts as `.md` files** ‚Äî `ats_skill.md` is the system prompt, stored separately from code. Prompt engineering iterations don't require Python changes.
- **Structured outputs** ‚Äî `model.with_structured_output(ResumeEvaluation)` forces Claude to return a typed Pydantic model, eliminating brittle JSON parsing.

## Running Locally

**Prerequisites:** Python 3.13+, [uv](https://docs.astral.sh/uv/), Node.js

```bash
# 1. Clone
git clone <repo-url>
cd bot_curriculum

# 2. Install dependencies
npm install -g @llamaindex/liteparse
uv sync

# 3. Environment variables
cp .env.example .env
# Add your ANTHROPIC_API_KEY to .env

# 4. Start
uv run uvicorn src.main:app --reload
```

Open http://localhost:8000

## API Reference

### `GET /health`

Returns server status. Used by the frontend to detect when the service has woken up from Render's free tier sleep.

**Response**
```json
{ "status": "ok" }
```

---

### `POST /evaluate`

Analyzes a resume and returns a structured ATS compatibility report.

**Rate limit:** 3 requests / minute per IP

**Request**

```
Content-Type: multipart/form-data

file: <PDF or DOCX file>
```

**Response `200 OK`**

```json
{
  "candidate_name": "Jane Smith",
  "overall_score": 74,
  "approved": false,
  "formatting_issues": [
    "Contact information is inside a table ‚Äî ATS may not parse it",
    "Two-column layout detected"
  ],
  "keywords_found": ["Python", "FastAPI", "REST API", "PostgreSQL"],
  "keywords_missing": ["Docker", "CI/CD", "Kubernetes", "Agile"],
  "recommendations": [
    "Add a Skills section listing missing keywords explicitly",
    "Switch to a single-column layout for better ATS parsing"
  ],
  "summary": "Solid technical background but missing several infrastructure keywords common in senior engineer job descriptions. Formatting issues may cause ATS parsers to misread sections."
}
```

**Error responses**

| Status | Condition |
|--------|-----------|
| `400` | Empty file |
| `422` | Could not extract text (e.g. scanned/image PDF) |
| `429` | Rate limit exceeded |
| `500` | Unexpected server error |

## Deploying to Render

The `render.yaml` file configures the service automatically:

```yaml
services:
  - type: web
    name: bot-curriculum
    runtime: python
    buildCommand: npm install -g @llamaindex/liteparse && pip install -e .
    startCommand: uvicorn src.main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: PYTHON_VERSION
        value: "3.13"
```

**Steps to deploy your own instance:**

1. Fork this repo
2. Create a new **Web Service** on [render.com](https://render.com) pointing to your fork
3. Render auto-detects `render.yaml` ‚Äî no manual config needed
4. Add `ANTHROPIC_API_KEY` and `FRONTEND_BASE_URL` in the Environment tab

**Note on free tier sleep:** Render's free tier spins down inactive services. The frontend polls `GET /health` before sending a CV and shows a wakeup message if the server is starting up ‚Äî no action required from the user.
